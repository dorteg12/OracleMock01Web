Introduction

About this Demo

This Demo is meant to provide a website that is accessible for the public internet and is resilient to potential outages. This means that users are able to access the website through a web browser and have a failover site in case the first site has a catastrophe. The failover site must be a second site that is physically distant from the first one. In addition, a user must be able to use a single URL and have access to 2 different sites. Hitting the mark is having a website that functions even when a data center goes down.

  • Create a Health Check
  • Create and attach Geolocation Steering Policy

Prerequisites

In order to run this workshop you need:

  • A Ubuntu Linux machine.
  • An OCI account with a compartment set up.
  • Subscription to two or more Regions

Make sure you have available resources and quota for:

  • 1 file storage systems.
  • 2 region subscriptions
  • 2 compute instances
  • 1 domain name
