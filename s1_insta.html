Introduction

Oracle Cloud Infrastructure Compute lets you provision and manage compute hosts, known as instances. You can launch instances as needed to meet your compute and application requirements. After you launch an instance, you can access it securely from your computer, restart it, attach and detach volumes, and terminate it when you're done with it. Any changes made to the instance's local drives are lost when you terminate it. Any saved changes to volumes attached to the instance are retained.

Be sure to review Best Practices for Your Compute Instance for important information about working with your Oracle Cloud Infrastructure Compute instance.

Creating a Instance

Estimated Time: 1 hour

Objectives

In this Lab, you will:

  • Create two compute instances each in a different region
  • Connect to the compute instances
  • Install NginX HTTP server

Prerequisites

  • An Oracle Cloud Account - please view the Getting started page for instructions on getting an Oracle Cloud Account
  • An Ubuntu open source operating system

Note: The Ubuntu system is need to access the compute instances since the instructions below use the Canonical Ubuntu image and not the defualt Oracle Linux image. Some of the commands may not work in Oracle Linux.

Task1: Create a Compute Instance

Oracle Cloud Infrastructure offers both Bare Metal and Virtual Machine instances:

  • Bare Metal- A bare metal compute instance gives you dedicated physical server access for highest performance and strong isolation.
  • Virtual Machine- A Virtual Machine (VM) is an independent computing environment that runs on top of physical bare metal hardware. The virtualization makes it possible to run multiple VMs that are isolated from each other. VMs are ideal for running applications that do not require the performance and resources (CPU, memory, network bandwidth, storage) of an entire physical machine.

An Oracle Cloud Infrastructure VM compute instance runs on the same hardware as a Bare Metal instance, leveraging the same cloud-optimized hardware, firmware, software stack, and networking infrastructure.

  1. Click the Navigation Menu in the upper left. Navigate to Compute, and select Instances.

  2. Then click Create Instance. We will launch a VM instance for this lab.

  3. The Create Compute Instance wizard will launch. Enter a name for the server. Click Edit on the Networking area of the page. Make sure your in the correct compartment

  4. Select change an image on the Image and shape section.

    You will be changing the image from Oracle Linux to Canonical Ubuntu. However you can choose any of images for thsi lab we will be changing it.

  5. Under the Networking section, select Create new virtual cloud network. You can give any name to the VCN. Make sure that it is being created in the correct compartment. In addition, make sure Assign a public IPv4 address is selected this will be need later.

  6. Download the Private Key and Public Key and save them on your device. The keys will be used to connect to the compute instance.

    Your Instance should look similar to the image shown below. Go ahead and select create.

  7. The Instance will start provisioning. Once it is done the icon will turn green. Make sure to save the Public IP address given since it will be needed later on.

Task2: Allowing HTTP connections to VCN

Oracle Cloud Infrastructure offers both Bare Metal and Virtual Machine instances:

  1. Click the Navigation Menu in the upper left. Navigate to Networking, and select Virtual Cloud Networks.

  2. Click on the VCN that was created for your compute instance in task 1. This will send you the the Virtual Cloud Network Details for that VCN.

    • Select the subnet for that VCN
    • Select the Security List being displayed

    You should be in Security List Details for the VCN as shown below.

  3. In the Security List Details page select Add Ingress Rules

    • For Source CIDR enter: 0.0.0.0/0
    • For Destination Port Range enter: 80
    • Add a Description if neccessary

    This will allow HTTP connections to your instance.

Subscribing to a Region

In order to complete scenario 01 you need to create another instance in a different region that is physically distant.

  • Select the region on the top rigth
  • Select Manage Regions
  • Subscribe to another region

You will then have to change into that subscribed region and create a compute instance in that region by following the same procedures given in task 1 and task 2.

Task3: Connection to the Instances and Installing NginX HTTP Server

Note: You must have the Private and Public Keys for each instance in two different folders. You will also need the Public IP for each instance

  1. To connect to the instance, open a Ubuntu terminnal and navigate to the folder in which the Private and Public key for your Instance are located. Run the following commands. 
                                 chmod 400 private_ssh_key
                             ssh -i private_ssh_key ubuntu@public_ip_address

    Note: For this case we use ubuntu since we choose the Ubuntu image anything else would use opc@public_ip_address

    You will be prompted with a continue line go ahead and enter yes. The following is picture of what you should see.

  2. You now have access to the compute instance. Now you need to install NginX into your Virtual Machine.

    In order to make it as simple as possible to create a website, you will be installing a LEMP web server.

    • L - Linux operating system
    • E - EngineX (Nginx)
    • M - MySQL (MariaDB)
    • P - PHP
    Enter the following command into your virtual machines terminal. 

                                sudo apt install nginx mariadb-server php-mysql php-curl php-gd php-zip php-fpm
  3. Ngnix is now installed but you still have give access to port 80 which is HTTP connections. To do so run the following commands shown below. The first one allows connections to port 80 and the second one will save the rule so it can stay active. 
                                sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 80 -j ACCEPT
                            sudo netfilter-persistent save
    You should be able to search up your IP address on a web browser and land on a Welcome to nginx page. You can edit what is being displayed by adding or editing files in the html directory. You can go to that directory by running the following command. 
                                cd /var/www/html/

  4. You now have a compute instance that is running a web server. In order to complete scenario 1 you have repeat the same procedures for your second instance located in a different region.

    Once that is complete, you should have to virtual machines running the same website that are accessed by their public ip addresses.
