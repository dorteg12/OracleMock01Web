Introduction

GEOLOCATION STEERING

Geolocation steering policies distribute DNS traffic to different endpoints based on the location of the end user. Customers can define geographic regions composed of originating continent, countries or states/provinces (North America) and define a separate endpoint or set of endpoints for each region.

For more information on Geolocation Steering.

Creating a Geolocation Steering Policy

Estimated Time: 30 minutes

Objectives

In this Lab, you will:

  • Create a Geolocation Steering Policy
  • Manage DNS Records

Prerequisites

  • An Oracle Cloud Account - please view the Getting started page for instructions on getting an Oracle Cloud Account

Task 1: Geolocation Steering Policy

Setting up a Geolocation Steering Policy can be done through the Oracle Console. Below are the steps to follow.

  1. Click the Navigation Menu in the upper left. Navigate to Networking, and select Traffic Management Steering policies under the DNS Management section.

  2. Select Create Traffic Management Steering Policy and a pop up will open. Select Geolocation Steering for Policy Type.
    • Enter a unique nae for Policy Name
    • TTL means Time To Live, it can be left as default or changed to what works best
  3. For the Answer Pool section you will need two pools. You can add another pool by clicking on the Additional Answer button. The following must be filled.
      Answer Pool 1
    • Unique Answer Pool Name
    • Unique Name
    • Type: A
    • Rdata: Public IP of the First Instance.
    • Select Eligible
      Answer Pool 2
    • Unique Answer Pool Name
    • Unique Name
    • Type: A
    • Rdata: Public IP of the Second Instance.
    • Select Eligible
  4. Click on Add Global Catch-all and as add the pools created in the last step.
  5. Under the Attach Health Check Select Existing and add the Health check created in Lab 3.
  6. Under the Attached Domain section do the following.
    • Choose a Sub domain
    • Choose the compartment being used
    • Select the zone created in Lab 2

    Note: The subdomain, under the selected zone, that the policy will be attached to. This is concatenated with the zone name to generate the full attached domain name. For example, a subdomain of "subdomain" and a zone name of "example.com" would create an attachment at "subdomain.example.com".

  7. Click on Create Policy. The TMSP will be created with information on the policy.

Task 2: Manage DNS Records

Setting the Domain Records using the attached Domain created in the steering policy

  1. In the steering policy details which has been created select Attached Domains under the Resources section. Copy the domain name shown under the Attached Domains section.
  2. Next go back to the name.com page and navigate to the domain created and click on Manage DNS Records
  3. Add the following Records.
      First Record
    • Type: A
    • Host: Paste Domain Name
    • Answer: Public IP of First Instance
      Second Record
    • Type: A
    • Host: Paste Domain Name
    • Answer: Public IP of Second Instance
  4. The instances are now linked. The website created can now be accessed by using the Domain Name on a browser. If the first instance were to fail, the second instance will automatically take over. Scenario 1 is now completed.
